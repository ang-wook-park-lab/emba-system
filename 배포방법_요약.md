# 📦 배포 방법 요약

## GitHub 사용 여부

### ✅ GitHub 사용 (권장)
**장점:**
- 자동 배포 (코드 푸시 시 자동 배포)
- 버전 관리
- 협업 용이
- 무료 호스팅 서비스와 쉽게 연동

**필요한 경우:**
- Netlify, Railway, Render 등 자동 배포 서비스 사용 시
- 코드 버전 관리가 필요한 경우

### ❌ GitHub 없이 배포
**가능한 방법:**
- 수동 파일 업로드
- 드래그 앤 드롭 배포
- FTP 업로드

## 📋 GitHub 없이 배포하는 방법

### 방법 1: Netlify 드래그 앤 드롭 (가장 간단)

#### 1단계: 프론트엔드 빌드
```bash
cd frontend
npm install
npm run build
```

#### 2단계: Netlify에 업로드
1. https://app.netlify.com 접속
2. "Add new site" → "Deploy manually"
3. `frontend/dist` 폴더를 드래그 앤 드롭
4. 환경 변수 설정: `VITE_API_URL=https://your-backend-url/api`

**장점**: GitHub 계정 불필요, 즉시 배포 가능

---

### 방법 2: Railway 수동 배포

Railway는 GitHub 연동이 주된 방법이지만, CLI를 사용할 수 있습니다:

#### 1단계: Railway CLI 설치
```bash
npm install -g @railway/cli
```

#### 2단계: Railway 로그인
```bash
railway login
```

#### 3단계: 프로젝트 초기화
```bash
cd kyungheewonwoo/backend
railway init
```

#### 4단계: 배포
```bash
railway up
```

---

### 방법 3: Render 수동 배포

Render는 주로 GitHub 연동을 사용하지만, Docker 이미지를 사용할 수 있습니다.

## 📝 GitHub 없이 배포하는 단계별 가이드

### Netlify 드래그 앤 드롭 배포

#### 1단계: 백엔드 배포 (Railway CLI 사용)

```bash
# Railway CLI 설치
npm install -g @railway/cli

# 로그인
railway login

# 프로젝트 초기화
cd kyungheewonwoo/backend
railway init

# 환경 변수 설정
railway variables set NODE_ENV=production
railway variables set JWT_SECRET=your-secret-key
railway variables set JWT_EXPIRE=7d

# 배포
railway up
```

#### 2단계: 프론트엔드 빌드

```bash
cd kyungheewonwoo/frontend
npm install
npm run build
```

#### 3단계: Netlify에 업로드

1. https://app.netlify.com 접속
2. 계정 생성 (이메일로도 가능, GitHub 불필요)
3. "Add new site" → "Deploy manually"
4. `frontend/dist` 폴더 드래그 앤 드롭
5. Site settings → Environment variables
6. `VITE_API_URL` 추가: `https://your-backend.railway.app/api`

## 💡 최종 추천

### GitHub 계정이 있다면
**GitHub 사용 + 자동 배포**를 강력히 권장합니다.

**이유:**
- 코드 변경 시 자동 배포
- 버전 관리로 이전 버전 복구 가능
- 협업 시 유용
- 대부분의 무료 호스팅 서비스가 GitHub 연동 지원

### GitHub 계정이 없다면
**Netlify 드래그 앤 드롭**을 사용하세요.

**단계:**
1. 프론트엔드 빌드 (`npm run build`)
2. `frontend/dist` 폴더를 Netlify에 드래그 앤 드롭
3. 환경 변수 설정
